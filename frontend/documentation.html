1. Abstract

SAD is an advanced web service meant to help System Administrators and Developers keep installs their packages easily and efficiently.SAD uses technologies such as REST API's to streamline communication between client and server.

2. Introduction

Both System Admins and Developers are in need of an easy way to monitor their software components, considering the fact that conflicts between packages might result in difficulty when it comes to troubleshooting issues, which then proves being time consuming. The idea is to bring an interface for Systems and their packagesâ€™ management, providing easy access to a broad range of repositories.

SAD use tool like dpkg to install your favorite packages.

The only thing that has to be done is choosing a platform and package name, so that SAd will generate and compile a script installing everything you need..

3. Overview

SAd Is based on modern Software Design techniques and uses an MVC architecture.

3.1 Technologies

We are going to use html, css and javascript for frontend, in scope of stylizing and creating.(html for document strucutre and css designing and javascript for frontend processing)

Python 2.7 will be used to create a REST API for the aplication. The deb packages will be downloaded from launchpad lib and mongoDB is going to store the packages information(names, dependencies and conflicts).

3.2 Routes and End-points

SAd uses a series of routes and end-points to deliver content to users. The following is a hierarchical view of the Routes:

3.2.1 Routes

  • /search/{pkg_name}
  • /package/{pkg_name}
  • /package/{pkg_name}/{pkg_version}
  • /package/{pkg_name}/{pkg_version}/{pkg_architecture}
  • /package/{pkg_name}/{pkg_version}/{pkg_architecture}/download

3.2.2 End-points

  • /checkout
  • /install
